### Slide 2-5 — "Progressive Disclosure: Three Levels"
**Type: Single concept (staged loading)**

> **Speaker note**: 「這是 Skill 真正解決的問題。不是減少工具數量，而是讓『何時載入什麼知識』變得有彈性。過去我們把所有 domain 的編排邏輯塞進 CLAUDE.md，等於把整本百科全書塞進工作記憶。現在 Skill 讓 AI 可以按需翻閱。」

> **Citation**:
> *"Progressive disclosure ensures only relevant content occupies the context window at any given time."*
> （漸進式揭露確保在任何時刻，只有相關內容佔用上下文窗口。）
>
> | Level | When Loaded | Token Cost |
> |-------|------------|------------|
> | **Level 1: Metadata** | Always (at startup) | ~100 tokens per Skill |
> | **Level 2: Instructions** | When Skill is triggered | < 5,000 tokens |
> | **Level 3: Resources** | As needed | Effectively unlimited |
>
> — [platform.claude.com/docs/en/agents-and-tools/agent-skills/overview](https://platform.claude.com/docs/en/agents-and-tools/agent-skills/overview)

```
Minimalist concept illustration, 1:1 square, pure black background.

Three horizontal layers, arranged top to bottom,
each progressively wider and brighter:

Top layer (narrowest, dim gray outline):
A tiny white tag icon. To the right: "L1: METADATA"
Far right annotation: "~100 tokens"
Below in small gray: "always loaded — name + description only"

Middle layer (medium width, white outline):
A white document icon. To the right: "L2: INSTRUCTIONS"
Far right annotation: "< 5K tokens"
Below in small gray: "loaded when triggered"

Bottom layer (widest, bright white fill):
A white folder icon with multiple files. To the right: "L3: RESOURCES"
Far right annotation: "unlimited"
Below in small gray: "loaded as needed"

A vertical dotted line on the left side connects all three layers,
with a downward arrow suggesting "deeper = heavier = later."

Between L1 and L2, a small lightning bolt icon (trigger moment).
Between L2 and L3, a small magnifying glass icon (on-demand lookup).

At the very top, small text: "PROGRESSIVE DISCLOSURE"
At the very bottom, small gray citation:
"Anthropic, 2025 — Agent Skills Specification"

Style: Jack Butcher / Visualize Value —
layered reveal, width = weight, depth = latency.
The staircase shape communicates staged loading.
Black background, white and gray elements.
```

### Slide 2-7 — "On-Demand Orchestration"
**Type: Single concept (the real answer)**

> **Speaker note**: 「所以 2-1 到 2-3 的三個問題，答案不是簡單的『模組化』— domain 早就模組化了。真正的答案是：Skill 讓驅動工具的編排邏輯從『永遠佔用記憶體』變成『按需調度』。這就像是一種不需要 embedding 的 context-level RAG。」

> **Citation**:
> Agent Skills Spec — *"Skills use progressive disclosure to manage context efficiently: (1) Discovery — agents load only the name and description... (2) Activation — the agent reads the full SKILL.md instructions into context. (3) Execution — the agent follows the instructions, optionally loading referenced files."*
> （技能使用漸進式揭露來高效管理上下文：(1) 發現 — Agent 只載入名稱和描述… (2) 啟動 — Agent 將完整的 SKILL.md 指令讀入上下文。(3) 執行 — Agent 遵循指令，按需載入參考檔案。）
> — [agentskills.io/what-are-skills](https://agentskills.io/what-are-skills)

```
Minimalist concept illustration, 1:1 square, pure black background.

Center composition showing a transformation:

Top: Three small white circles labeled "Bloat" "Drift" "Conflict"
(same as original 2-4, the three problems).

Three thin white lines descend from these circles
and converge into a central mechanism —
NOT a single word, but a visual "switchboard":

Center: A white rounded rectangle containing three toggle switches
arranged vertically. Each switch has an "off" position (left, gray)
and an "on" position (right, white).
The switches are labeled:
  "fire-safety" — ON (white)
  "parking" — OFF (gray)
  "smoke" — OFF (gray)

This represents: only the relevant workflow is "switched on" at any moment.

Below the switchboard, a single bold phrase:
"ON-DEMAND ORCHESTRATION"

Below that, in smaller gray text:
"不是減少工具，是讓編排按需載入"
("Not fewer tools — orchestration loaded on demand")

Style: Jack Butcher / Visualize Value —
the switchboard metaphor replaces the vague "MODULAR" word,
toggles as the core visual showing selective activation.
Black background, white and gray elements.
```
